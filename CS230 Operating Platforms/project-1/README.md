# CS 230 Project One - Design Patterns: Singleton & Iterator

Java app implementing singleton (one GameService instance) and iterator (check duplicate game names). UML diagram required.

## Lessons Learned

- **[Java, Singleton]:** Private ctor + static `getInstance()` guarantees one instance. Essential for shared state (DB, config).
- **[Java, Iterator]:** Java `Iterator` interface for name uniqueness checks. Built-in iteration safer than raw loops.
- **[Java, UML]:** Diagramming before code clarifies class relationships. Singleton stereotyped as «singleton».
- **[Java, Eclipse FIXME]:** IDE task tags (`FIXME`/`TODO`) navigate codebase efficiently. Team workflow tool.
